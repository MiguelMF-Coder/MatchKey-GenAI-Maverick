01 · Ingeniería del Dato — MatchKey (TFG Business Analytics)¶

Objetivo del TFG que aborda: rúbrica de Ingeniería del Dato — inventario, calidad, limpieza y exploración auditable de los datasets de MatchKey.

Qué hace este notebook

  1. Inventario de datasets (origen, tamaño, esquema).
  2. Diccionario de variables (tipo, nulos, únicos, ejemplo).
  3. Calidad: nulos y duplicados.
  4. Limpieza documentada (antes / después).
  5. EDA estadístico (distribuciones).
  6. Auditoría de calidad (honestidad sobre los datos).
  7. Hallazgos clave.

Entradas: data/matchkey/*.csv (export reproducible desde los JSON de producción, generado por analytics/export_data.py) + backend/.../skills_dictionary.json + (si está disponible) data/esco/.

Salidas: data/clean/*.csv (datasets limpios) y figuras en notebooks/figures/.

⚠️ Honestidad sobre los datos (ver CLAUDE.md):

  • category_llm es silver-standard (etiqueta generada por un LLM leyendo la oferta), no anotación humana.
  • En el backend, skills_extractor usa un diccionario base mínimo y values_fit/team_fit son heurísticos; aquí se auditan, no se presentan como resultados.
  • La fuente de verdad de las skills de vacante son los arrays del JSON de scraping (tabla job_skills de la BD estaría casi vacía), por eso trabajamos sobre el export plano.

def data_dictionary(df: pd.DataFrame, table: str) -> pd.DataFrame:
    """Para cada columna: tipo, nº y % de nulos, nº de valores únicos y un ejemplo."""
    rows = []
    for c in df.columns:
        s = df[c]
        ejemplo = s.dropna().iloc[0] if s.notna().any() else None
        if ejemplo is not None and len(str(ejemplo)) > 60:
            ejemplo = str(ejemplo)[:60] + "..."
        rows.append({
            "tabla": table, "columna": c, "dtype": str(s.dtype),
            "n_nulos": int(s.isna().sum()), "pct_nulos": round(100 * s.isna().mean(), 1),
            "n_unicos": int(s.nunique(dropna=True)), "ejemplo": ejemplo,
        })
    return pd.DataFrame(rows)

4. Limpieza documentada (antes / después)¶

Transformaciones aplicadas (todas trazables y reversibles desde el export):

  • vacantes: normalización de category_llm (strip; vacías/nulas → "Sin categoría"), flag tiene_salario, experiencia_anios numérica.
  • candidatos: years_experience numérica, n_skills_total.
  • candidate_skills / job_skills: normalización de la skill (strip().lower()) y deduplicado.

La celda siguiente aplica las transformaciones y la figura muestra el impacto cuantitativo antes/después.

vac = data["vacantes"].copy()
vac["category_llm"] = vac["category_llm"].astype("string").str.strip().replace({"": pd.NA})

# (c) Outliers por la regla de Tukey (1,5·IQR)
def iqr_outliers(s):
    s = s.dropna()
    if len(s) == 0:
        return 0, 0
    q1, q3 = s.quantile(0.25), s.quantile(0.75)
    iqr = q3 - q1
    lo, hi = q1 - 1.5 * iqr, q3 + 1.5 * iqr
    return int(((s < lo) | (s > hi)).sum()), len(s)

print("\nOutliers (regla de Tukey 1,5·IQR):")
for col in ["experiencia_anios", "salary_min", "n_skills_nice", "n_tech_stack"]:
    n_out, n_tot = iqr_outliers(num_vac[col])
    print(f"  vac.{col}: {n_out}/{n_tot}" + (f" ({100*n_out/n_tot:.1f}%)" if n_tot else " (sin datos)"))
for col in ["years_experience", "n_skills_total"]:
    n_out, n_tot = iqr_outliers(num_cand[col])

7. Hallazgos clave¶

  • Volumen: 100 candidatos, 236 vacantes, 5.411 cursos, 82 empresas. Datasets pequeños → los modelos deben reportar resultados con cautela (intervalos, no solo puntuales).
  • category_llm muy desbalanceada (clase mayoritaria ~58%; varias clases con ≤4 ejemplos): el baseline trivial ya es alto y el macro-F1 estará dominado por clases minoritarias. Decisión pendiente en el Modelo 3 (agrupar clases raras vs. reportar F1 por clase).
  • category_jobs (Adzuna) ~70% "Unknown" → no usable como etiqueta; category_llm es la única viable.
  • El diccionario base de skills del backend es mínimo → no es un extractor real; las skills reales provienen del OCR (candidatos) y del LLM (vacantes).
  • values_fit/team_fit son heurísticos en el backend → se auditarán por contraste, no se presentan como verdad.
  • Datasets limpios escritos en data/clean/ y figuras en notebooks/figures/ para la memoria.
